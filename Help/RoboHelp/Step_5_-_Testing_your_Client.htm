Client Development Tutorial

Testing a Client requires much patience, and unfortunately cannot be automated like a Server.

Launch Capability

  1. Compile and build your newly developed client.

  2. Launch the Reference Server and then connect the client to the server.

  3. Right-click on the project and choose "Start new instance" from the "Debug" menu.

  4. The application should launch successfully.

Subscribe

  1. Once the client is connected you will use the node browser to navigate to any node here:
    CTT -> Dynamic -> All Profiles -> Scalar

  2. Select a child node such as Int16, by simply clicking on it once.

  3. Now click on our "Subscribe" menu item, which is in the "Server" menu.

  4. The new "SubscriptionOutput" window should open and you should see data-changes update the screen about once per second.

Test with UA CTT (Compliance Test Tool)

  1. Lastly, but most importantly you need to test your client for compliance.

  2. Download and install the latest UA CTT (Compliance Test Tool) from the OPC Foundation website.

  3. The UA CTT contains product documentation (similar to this) which describes this powerful tool, and also contains tutorials which demonstrate how to test a client.

  4. Open the product documentation and navigate to the following page:
    Compliance Testing -> Client Testing -> Introduction.
