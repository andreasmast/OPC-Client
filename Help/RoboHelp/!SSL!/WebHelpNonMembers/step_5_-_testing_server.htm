Server Development Tutorial

In this step we will test our server as follows:

  • Launch capability

  • Identification (application name and vendor name)

  • Reading a value

  • Writing to a value

  • Compliance Testing

Launch capability

  1. Compile and build your newly developed server.

  2. Right-click on the project and choose "Start new instance" from the "Debug" menu.

  3. The application should launch successfully. You may be prompted by Windows Firewall to permit traffic to/from this server - you are encouraged to allow this behavior.

Identification (application name and vendor name)

  1. Once the application is running you will see the application name clearly stated at the top of the window.

  2. Connect a Client to the Server and then navigate to the following node within the address space and then read the value:
    Objects -> Server -> ServerStatus -> BuildInfo -> ProductName

  3. Read the value of the following node:
    Objects -> Server -> ServerStatus -> BuildInfo -> ManufacturerName

Reading a value

  1. Connect a client to the server and then navigate and read the following node:
    Objects -> CTT -> MyFolder -> MyNode.

Writing to a value

  1. While connected to the "MyNode" you will elect to write a value to the node.

  2. Upon writing to the node, your UA Server will display a modal messagebox window to the screen showing you the value you sent. Simply dismiss this window.

Compliance Testing

  1. Lastly, but most importantly you need to test your server for compliance.

  2. Download and install the latest UA CTT (Compliance Test Tool) from the OPC Foundation website.

  3. The UA CTT contains product documentation (similar to this) which describes this powerful tool, and also contains tutorials which demonstrate how to test a server.

  4. Open the product documentation and navigate to the following page:
    Compliance Testing -> Server Testing -> Introduction.
